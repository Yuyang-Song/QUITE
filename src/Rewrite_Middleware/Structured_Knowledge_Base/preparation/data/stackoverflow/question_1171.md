## Answers
### Answer ID: 62028910
<p>In order to achieve that on Django 2.2 <a href="https://docs.djangoproject.com/en/2.2/howto/custom-lookups/#a-simple-transformer-example" rel="nofollow noreferrer">you'll need to register two new <code>JSONField</code> transforms/lookups</a> since <a href="https://github.com/django/django/commit/e9a0e1d4f6ecfc4227acff74e1f56f288a0b30aa" rel="nofollow noreferrer">support for conditional expressions was only added in the upcoming 3.1 release</a>.</p>

<p>You'll first want to register lookups for <code>JSONField</code> key accesses</p>

<pre class="lang-py prettyprint-override"><code>from django.db import models
from django.db.models.lookups import Lookup
from django.contrib.postgres.fields.jsonb import (
    KeyTransform, KeyTransformTextLookupMixin
)

@KeyTransform.register_lookup
class KeyTransformIsInteger(KeyTransformTextLookupMixin, Lookup):
    lookup_name = 'is_int'
    prepare_rhs = False

    def as_sql(self, compiler, connection):
        key_expr = KeyTransform(
            self.lhs.key_name, *self.lhs.source_expressions, **self.lhs.extra
        )
        key_sql, key_params = self.process_lhs(
            compiler, connection, lhs=key_expr
        )
        lhs_sql, lhs_params = self.process_lhs(compiler, connection)
        rhs_sql, rhs_params = self.process_rhs(compiler, connection)
        sql = "(jsonb_typeof(%s) = %%s AND mod(%s::numeric, %%s) = %%s) IS %s" % (
            key_sql, lhs_sql, rhs_sql
        )
        params = [
            *key_params, 'number',
            *lhs_params, 1, 0,
            *rhs_params,
        ]
        return sql, params

@KeyTransform.register_lookup
class KeyTransformIntegerGt(KeyTransformTextLookupMixin, Lookup):
    lookup_name = 'int_gt'
    prepare_rhs = False

    def as_sql(self, compiler, connection):
        lhs_sql, lhs_params = self.process_lhs(compiler, connection)
        rhs_sql, rhs_params = self.process_rhs(compiler, connection)
        sql = "%s::int &gt; %s" % (lhs_sql, rhs_sql)
        params = [*lhs_params, *rhs_params]
        return sql, params
</code></pre>

<p>Once this is done you should be able to define your constraint like</p>

<pre class="lang-py prettyprint-override"><code>CheckConstraint(
    check=Q(
        linkage__root__in_sub_project__is_int=True,
        linkage__root__in_sub_project__int_gt=0,
    ),
    name='asp_sub_project_positive_integer',
)
</code></pre>

<p>Once you're on Django 3.1 you should be able to pass a <code>RawSQL</code> directly to <code>CheckConstraint.check</code> as long as it has an <code>output_field = models.BooleanField()</code>.</p>

<pre class="lang-py prettyprint-override"><code>RawSQL("""
   jsonb_typeof(linkage-&gt; 'root' -&gt; 'in_sub_project') is not distinct from 'number'
   and (linkage-&gt;'root'-&gt;&gt;'in_sub_project')::numeric % 1 = 0
   and (linkage-&gt;'root'-&gt;&gt;'in_sub_project')::numeric &gt; 0
""",
    output_field=models.BooleanField()
)
</code></pre>
