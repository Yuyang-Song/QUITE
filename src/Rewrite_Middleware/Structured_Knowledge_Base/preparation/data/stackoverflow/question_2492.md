# How to generate &quot;select count(*) ...&quot; using CriteriaQuery
[Link to question](https://stackoverflow.com/questions/37012587/how-to-generate-select-count-using-criteriaquery)
**Creation Date:** 1462302993
**Score:** 1
**Tags:** java, oracle-database, hibernate, jpa, criteriaquery
## Question Body
<p>I wrote the following code to get the count of "<em>ExampleEntity</em>":</p>

<pre><code>cb = entityManager.getCriteriaBuilder();
CriteriaQuery&lt;Long&gt; cq = cb.createQuery(Long.class);
root = cq.from(ExampleEntity);
cq.select(cb.count(root));
return entityManager.createQuery(cq).getSingleResult();
</code></pre>

<p><strong>Generated sql:</strong>
Hibernate: select count(<strong>exampleen0_.id</strong>) as col_0_0_ from EXAMPLEENTITY exampleen0_</p>

<p>But, for performance reasons (using a Oracle 11g database),I need to generate the following sql:  </p>

<p><strong>Desired sql:</strong> select count(*) as col_0_0_ from EXAMPLEENTITY exampleen0_</p>

<p>It's quite simple to do creating JPQL queries, but I would have to rewrite a lot of existing code for filters.</p>

<p>How can I generate "count(*)" instead of "count(exampleen0_.id)" using CriteriaQuery?</p>

## Answers
### Answer ID: 39729227
<p>It works in Oracle...</p>

<pre><code>BigDecimal count = (BigDecimal) manager.createNativeQuery(query).getSingleResult();
</code></pre>

### Answer ID: 37013198
<p>It's extremely hard to force the hand of the SQL generated by Hibernate. Instead, I would consider writing a native query.</p>

<pre><code>Query query = entityManager.createNativeQuery(
        "SELECT COUNT(*) FROM EXAMPLEENTITY", Long.class);
return query.getSingleResult();
</code></pre>

<p><strong>Note:</strong> Native queries can get messy when using more complex logic, but this particular one complies to the ANSI standard and should run without issue against any commonly used DB without having to worry about any db-specific syntax issues.</p>
