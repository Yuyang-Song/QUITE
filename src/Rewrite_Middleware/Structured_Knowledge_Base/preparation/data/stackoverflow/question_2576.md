**Tags:** php, activerecord, yii, orm, propel
## Question Body
<p>Is it possible to use such ORMs as Propel or Yii's ActiveRecord without database connection?</p>

<p>There's a Yii2 project in which I need PHP not to communicate to database directly but via the UNIX socket. Best case scenario is extending/rewriting Yii's <code>Model::class</code> and being able to execute any CRUD method in a way that it'd generate a plain SQL and send it in the socket. I've looked through Yii's <code>Model::class</code>, <code>Query::class</code> and <code>QueryBuilder::class</code> source codes and it looks like they always need an instance of database connection.</p>

<p>Right now, for me, it seems like it's not even possible.</p>

<p>A worse (but still good, though) scenario is using Propel. I know it has a <code>toString()</code> method when building queries, but, as far as I know, it's not possible to use it in methods of models generated by Propel (like <code>$user-&gt;getPhotos()</code>, etc.).</p>

<p>Now I'm wondering whether it's even possible and, perhaps, there different kind of tools for this?</p>
