## Answers
### Answer ID: 74884999
<p>Another cause of this issue is if you remove all $_SESSION variables using the unset() function before setting new values. Any $_SESSION variable set after that will <strong>not</strong> hold its value across controllers. Using the unset() function in this way disables the registration that holds session values, even if your CI_Version is up to date.</p>
<p>Instead of doing</p>
<pre><code>unset($_SESSION);
$_SESSION['test'] = &quot;Testing&quot;;
</code></pre>
<p>use this to free up all session variables before setting new ones:</p>
<pre><code>session_unset();
$_SESSION['test'] = &quot;Testing&quot;;
</code></pre>
<p>More information is here: <a href="https://www.php.net/manual/en/function.session-unset.php" rel="nofollow noreferrer">https://www.php.net/manual/en/function.session-unset.php</a></p>

### Answer ID: 56138369
<p>Try creating your session directory based on your current project paths and set "sess_save_path" to a relative path. You can use one of the predefined relatives.</p>

<pre><code>FCPATH   -&gt; '/'
BASEPATH -&gt; '/system/'
APPPATH  -&gt; '/application/'
</code></pre>

<p>For example</p>

<pre><code>$config['sess_save_path'] = BASEPATH . 'sess';
</code></pre>

<p>sets path to be</p>

<pre><code>&lt;your-codeigniter-start-folder&gt;/system/sess
</code></pre>
