## Answers
### Answer ID: 21436997
<p><strong>Remove session_start() from your code. CodeIgniter will handle session initialization.</strong></p>

<p>Because CodeIgniter uses a named session, you would need to specify it first to use PHP native sessions:</p>

<pre><code>session_name('ci_session');
session_start();
</code></pre>

<p>But you shouldn't be doing this. The whole reason CodeIgniter <a href="http://ellislab.com/codeigniter%20/user-guide/libraries/sessions.html" rel="nofollow">has a Session class</a> is to act as an interface between native PHP and CodeIgniter sessions (especially when used with a database). You shouldn't have to use PHP session methods to begin with.</p>

<p>Actually, by running <code>session_start()</code>, you might be switching from the CodeIgniter session to the PHP native session, but you are still using CodeIgniter session calls, so nothing is saved to the PHP session you are working in.</p>

### Answer ID: 21437187
<p>I see two problems. First you set session data wrong (<a href="http://ellislab.com/codeigniter/user-guide/libraries/sessions.html" rel="nofollow">manual</a>):</p>

<p>Change this:</p>

<pre><code>$this-&gt;session-&gt;set_userdata('logged_in', $sess_array);
</code></pre>

<p>Into this:</p>

<pre><code>$this-&gt;session-&gt;set_userdata($sess_array);
</code></pre>

<p>Then read session data like this:</p>

<pre><code>$this-&gt;session-&gt;userdata('id');
$this-&gt;session-&gt;userdata('username');
</code></pre>

<p>Good Luck!</p>
