**Tags:** php, ajax
## Question Body
<p>I'm running a website that uses AJAX to populate products from the database. Now I want to make this site crawlable for searchenginge's. I'v read this post on google webmasters <a href="https://developers.google.com/webmasters/ajax-crawling/" rel="nofollow">Making AJAX Applications Crawlable</a>. But I have no clue how to make this work in my application. I just don't see how to apply this on my application without rewriting big parts of the code. Hopefully someone of you guys might give me some tips.</p>

<p><strong>My application</strong>
<br>
URL: <code>www.mysite.com/productcategory/category1/</code>
<br>This URL show products from category1 and a AJAX call creates the HTML. There are no special URL parameters. This is done because there is pagination. Clicking on the next page executes the same AJAX script but now it returns products for the next page. The URL will be modified (with jQuery history) to: 
<br><code>www.mysite.com/productcategory/category1/?page=2</code></p>

<p>Clicking on the <code>next page</code> button a script called <code>get_products.ajax.php</code> and all parameters (like page=2) are send by a <code>POST</code>. So the call to the <code>get_products.ajax.php</code> is always the same, no parameters in the URL. Also a lot of information is stored in sessions to limit the number of database queries. So the list of products where the <code>get_product.ajax.php</code> is looking in, is a JSON string in a session.</p>

<p><strong>Crawlable</strong>
<br>To make the site crawlable, I need to create a snapshot of the HTML generated by the AJAX call. This all make's sence, but I have no clue of how I can show this content to Google in stead of the default AJAX content.</p>

<p>I'm staring at this issue all morning but I just don't see where I need to change things in my code...</p>

## Answers
### Answer ID: 28321402
<p>The simplest way is adding <code>&lt;meta name="fragment" content="!"&gt;</code> between <code>&lt;head&gt;&lt;/head&gt;</code> tags on all your pages where you are using AJAX calls. In your case this means you should add this meta tag to the template which is used for rendering <code>www.mysite.com/productcategory/category1/</code>.</p>

<p>This will cause crawler will ask your page with <code>?_escaped_fragment_=</code> argument. Your task is to handle request with this parameter and return rendered page to the crawler.</p>

<p>For rendering you can use PhantomJS, <a href="https://prerender.io/" rel="nofollow">https://prerender.io/</a> or something else.</p>
