# The LINQ expression &#39;p&#39; could not be translated when building custom expression

**Score:** 0

## Question Body
<p>I have a simple Order class</p>
<pre><code>public abstract class Entity
{
   public int Id { get; set; }
}

public class Order : Entity
{
    public string Description { get; set; }
    public string DeliveryAddress { get; set; }
    public decimal Price { get; set; }
    public int Priority { get; set; }
}
</code></pre>
<p>And I have some dynamically made filters that should be converted to Expressions and queried against a database.</p>
<pre><code>public class Filter&lt;T&gt;
    where T : Entity
{
    public Expression PropertySelector { get; set; }
    public Operator Operator { get; set; }
    public dynamic FilteringValue { get; set; }
}

public enum Operator
{
    Equal = 0,
    GreaterThan = 1,
}
</code></pre>
<p>Now, let's say this is an entry point and a place where filters are defined (and where the response should be evaluated).</p>
<pre><code>public IEnumerable&lt;Order&gt; GetByFilter()
{
    var filters = new List&lt;Filter&lt;Order&gt;&gt;()
        {
            new()
            {
                PropertySelector = ((Expression&lt;Func&lt;Order, int&gt;&gt;) (p =&gt; p.Priority)).Body,
                Operator = Operator.GreaterThan,
                FilteringValue = 1,
            },
            new()
            {
                PropertySelector = ((Expression&lt;Func&lt;Order, string&gt;&gt;) (p =&gt; p.Description)).Body,
                Operator = Operator.Equal,
                FilteringValue = &quot;Laptop&quot;,
            }
        }

    IQueryable&lt;Order&gt; queryableOrders = _orderRepository.QueryAll();
    queryableOrders = _orderRepository.QueryByCustomerFilter(queryableOrders, filters);

    return queryableOrders.AsEnumerable();
}
</code></pre>
<p>I pass filters to some method in repository...</p>
<pre><code>public IQueryable&lt;T&gt; QueryByCustomerFilter(IQueryable&lt;T&gt; source, List&lt;Filter&lt;T&gt;&gt; filters)
{
    var entityType = source.ElementType;
    var predicate = PredicateBuilder.GetFilterPredicate(entityType, filters);

    return source.Where(predicate);
}
</code></pre>
<p>And finally, I have a PredicateBuilder that is responsible to generate predicate based on specified filters...</p>
<pre><code>public static class PredicateBuilder
{
    public static Expression&lt;Func&lt;T, bool&gt;&gt; GetFilterPredicate&lt;T&gt;(Type entityType, List&lt;Filter&lt;T&gt;&gt; filters)
        where T : Entity
    {
        var entity = Expression.Parameter(entityType, &quot;p&quot;);
        var buildingBlocks = new List&lt;Expression&gt;();

        foreach (var filter in filters)
        {
            var left = filter.PropertySelector;
            var right = Expression.Constant(filter.FilteringValue);

            var buildingBlockExpression = filter.Operator switch
            {
                Operator.Equal =&gt; Expression.Equal(left, right),
                Operator.GreaterThan =&gt; Expression.GreaterThan(left, right),
                _ =&gt; throw new ArgumentOutOfRangeException(nameof(filter.Operator)),
            };

            buildingBlocks.Add(buildingBlockExpression);
        }

        var customFilterExpression = buildingBlocks.Aggregate(Expression.AndAlso);
        return Expression.Lambda&lt;Func&lt;T, bool&gt;&gt;(customFilterExpression, entity);
    }
}
</code></pre>
<p>But when I run this code I get:</p>
<p><em><strong>System.InvalidOperationException</strong>: 'The LINQ expression 'p' could not be translated. Either rewrite the query in a form that can be translated, or switch to client evaluation explicitly by inserting a call to 'AsEnumerable', 'AsAsyncEnumerable', 'ToList', or 'ToListAsync'.</em></p>
<p>However, <strong>here is what confuses me</strong>. If I put the next line of the code in QueryByCustomerFilter method, I can clearly see that there is no difference between the expression written for that line of the code and the expression that is generated by the code based on filters.</p>
<pre><code>var testQueryable = Context.Orders.Where(p =&gt; p.Priority &gt; 1 &amp;&amp; p.Description == &quot;Laptop&quot;).AsQueryable();
</code></pre>
<p>SQL query for both expressions is identical and I can't see any difference.</p>
<pre><code>&quot;SELECT [o].[Id], [o].[DeliveryAddress], [o].[Description], [o].[Price], [o].[Priority]\r\nFROM [Orders] AS [o]\r\nWHERE ([o].[Priority] &gt; 1) AND ([o].[Description] = N'Laptop')&quot;
</code></pre>
<p>Finally, <strong>the most confusing part</strong> is that if I do</p>
<pre><code>testQueryable.ToList()
</code></pre>
<p>before the original query is evaluated everything will work as expected. So, <strong>BOTH of the expressions are successfully translated</strong> and I am able to get expected results.</p>
<p>So, what's happening here? Why the original expression can't be translated and how are two queryables from the example connected between each other?</p>

## Answers
### Answer ID: 70378829
<p>Parameter that you are creating via <code>var parameterReplacerVisitor = new ParameterReplacerVisitor(entity);</code> and the ones in your expressions in <code>PropertySelector</code> are different despite having the same names. You need to replace incoming with the created one. For example:</p>
<pre><code>class ParameterReplacerVisitor : ExpressionVisitor
{
    private readonly ParameterExpression _param;

    public ParameterReplacerVisitor(ParameterExpression expression)
    {
        _param = expression;
    }

    protected override Expression VisitParameter(ParameterExpression node) =&gt; _param;
}
</code></pre>
<pre><code>public static Expression&lt;Func&lt;T, bool&gt;&gt; GetFilterPredicate&lt;T&gt;(Type entityType, List&lt;Filter&lt;T&gt;&gt; filters)
    where T : Entity
{
    var entity = Expression.Parameter(entityType, &quot;p&quot;);
    var buildingBlocks = new List&lt;Expression&gt;();

    foreach (var filter in filters)
    { 
        var parameterReplacerVisitor = new ParameterReplacerVisitor(entity);
        
        var left = filter.PropertySelector;
        left = parameterReplacerVisitor.Visit(left);
        ....
     }
}
</code></pre>
