# CakePHP 3 + ORM Query builder and column name escape
[Link to question](https://stackoverflow.com/questions/48631083/cakephp-3-orm-query-builder-and-column-name-escape)
**Creation Date:** 1517863158
**Score:** 5
**Tags:** sql, cakephp, cakephp-3.x, quoting, reserved-words
## Question Body
<p>A've already rewrite some application from CakePHP 2 to CakePHP 3. I have some structure in database with column: key (autogenerated key-string). In mysql key is a keyword, so when I write an SQL Query I have to escape it as </p>

<pre><code>INSERT INTO table (`key`) VALUES (....)
</code></pre>

<p>Unfortunately when I try to save Entity I receive an error: </p>

<pre><code>[42000][1064] You have an error in your SQL syntax; check the manual that corresponds to your MariaDB server version for the right syntax to use near 'key, 
</code></pre>

<p>That error is triggered in Cake\ORM\Table in _insert() method on that place:</p>

<pre><code>$statement = $this-&gt;query()-&gt;insert(array_keys($data))
        -&gt;values($data)
        -&gt;execute();
</code></pre>

<p>Any suggestions how to avoid that situation? CakePHP version 3.5.11</p>

## Answers
### Answer ID: 48631570
<p>Either rename the column to avoid that conflict, or enable CakePHPs automatic identifier quoting, either globally in the database connection configuration via the <code>quoteIdentifiers</code> option:</p>

<pre><code>// in config/app.php

'Datasources' =&gt; [
    'default' =&gt; [
        // ...
        'quoteIdentifiers' =&gt; true,
    ],

    // ...
]
</code></pre>

<p>or only for that specific operation by either toggling the drivers auto quoting flag on the fly:</p>

<pre><code>$driver = $this-&gt;getConnection()-&gt;getDriver();
$autoQuouting = $driver-&gt;isAutoQuotingEnabled();
$driver-&gt;enableAutoQuoting(true);

$this-&gt;query()/* ... */;

$driver-&gt;enableAutoQuoting($autoQuouting);
</code></pre>

<p>or by manually quoting the name and passing it in a quoted fashion:</p>

<pre><code>$connection = $this-&gt;getConnection();
$quotedColumnName = $connection-&gt;quoteIdentifier($columnName);
// ...
</code></pre>

<p>this is currently compatible with auto quoting, ie the quoted name wouldn't be quoted twice if auto auoting would be enabled, but I'm not sure if this is actually a promised behavior!</p>
