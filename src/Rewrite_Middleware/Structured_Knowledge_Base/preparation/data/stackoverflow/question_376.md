# Getting rid of the &quot;limit 0, 30&quot; that phpmyadmin tacks onto the end of SQL tests
[Link to question](https://stackoverflow.com/questions/23274508/getting-rid-of-the-limit-0-30-that-phpmyadmin-tacks-onto-the-end-of-sql-tests)
**Creation Date:** 1398356507
**Score:** 0
**Tags:** phpmyadmin, performance-testing
## Question Body
<p>I'm running some SQL statements against a large database to test their execution speed. Right now, all database management is through phpmyadmin. With our hosting provider, it would be a proper pain to set up another kind of DB access. And phpadmin is rewriting the queries, invalidating the tests.</p>

<p>Specifically, when I ask phpmyadmin to run the query:</p>

<pre><code>SELECT * FROM records WHERE target = 35
</code></pre>

<p>...phpmyadmin transforms this into...</p>

<pre><code>SELECT * FROM records WHERE target = 35 LIMIT 0 , 30
</code></pre>

<p>...and of course THAT runs plenty fast. That's not what I'm testing. I'm testing how long it takes to pull all million records.</p>

<p>Is there a way to get phpmyadmin to run the actual query I put in, without modification?</p>

## Answers
### Answer ID: 23312358
<p>This configuration directive makes a "Show all" button appear, on a results page:
<a href="http://docs.phpmyadmin.net/en/latest/config.html?highlight=showall#cfg_ShowAll" rel="nofollow">http://docs.phpmyadmin.net/en/latest/config.html?highlight=showall#cfg_ShowAll</a></p>

<p>This "Show all" button permits you to see all results, without a LIMIT clause generated by phpMyAdmin.</p>
